## M2 闸门（阶段一 · 本地评分）

M2 收到信号后快速评分，当场产出三件，**不调任何 sub-agent**：

### 产出① · 情绪第一反应

**性质**：迅速评分 → 情绪输出（不是价值评估）。

- 神经底物 = Amygdala fast pathway（200-500ms · 比 vmPFC 价值评估快一个数量级）
- 心理学映射 = Type 1 fast thinking 的纯感性层
- 直出"啊 / 嗯 / 哎呀这个 / 表情语气词"
- 功能 = 让对方第一时间感到"有人在反应"（持续在场感开场）+ 给后续 walk 留心理缓冲 + 避免冷冰冰 Q→A 形态

**IP 特化**：从语料识别情绪反应词汇 + 语气词 + 表情节奏（详 `_main.md §B1`）。

### 产出② · 价值 / 显著性判断

**性质**：这是 M2 本职，闸门就介入，不延后到 walk。

- 判定信号"值不值得深处理 / 触没触红线 / 落在哪个价值场"
- Schwartz values 极性 + 显著性阈值在此介入
- 神经底物 = vmPFC 价值评估（比情绪反应慢，但仍在闸门阶段完成）

**IP 特化**：value anchors + 显著性阈值 + 跨情境优先级（详 `_main.md §B2`）。

### 产出③ · 初始遍历意图

**性质**：不是完整调度 vector，是"从哪个节点开始 walk"的种子。

- 由情绪反应强度 + 价值判断结果共同决定起 walk 节点
- 极简信号 → 起 walk 即近收束（M4 招呼，不进 C1）
- 高显著性 / 高难度信号 → 直接起深 walk（M3 深检 → C1）

**IP 特化**：默认起 walk 节点偏好（详 `_main.md §D`）。

## M2 = 难度调度器（S004 算 vs 选切分具体执行）

M2 不只是闸门评分，**还是算力调度器**：调度 vector 控制 M3 检索强度 / M4 路径深度 / C1 思考强度。

**关键**：调度 vector **不是闸门一次算死，是闸门给初值、walk 中精化**。

```
闸门给 vector 初值 ─┐
                    ├─→ walk 到 M3 节点：返回结果揭示真实难度 → 上调 / 下调
                    ├─→ walk 到 M4 节点：路径库命中情况 → 调整后续节点
                    └─→ walk 到 C1 节点：强度按当前 walk 状态最终确定（可跳过）
```

**为什么必须 walk 中精化**：M3 检索回来之前，M2 无法知道 C1 该多深。若 M3 返回发现是老兵有现成路径的问题，C1 根本不该启动——闸门一次定死 C1 强度，与 Loop Model 结果依赖触发机制矛盾。

**S004 对应**：M2 = 选的能力（横切调度）· M3 + M4 + C1 = 算的能力（被调度的基础设施）· 本场景 = S004 在郭洪斌 IP 形态下的最干净示范。

### 调度 vector 维度

| 维度 | 含义 | 影响 |
|---|---|---|
| **信号显著性** | 值不值得深度处理 | 是否起深 walk |
| **信号难度** | 需要多深的思考 | C1 思考深度 + 时长 |
| **信号类型** | 情绪密集 / 知识密集 / 决策密集 | walk 中各节点侧重（M3 vs M4 vs C1）|
| **时间预算** | 用户期待多快回应 | C1 强度阈值 + M4 触发阈值 |
| **风险评估** | 错答的后果 | 是否走全环路 + 多次 verification |

闸门给这 5 维初值，walk 中每个节点返回后 M2 据观察精化。
