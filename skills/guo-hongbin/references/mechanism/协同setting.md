## 渐进序 = 典型 walk 轨迹（不是刚性时间轴）

渐进序描述的是**中高难度信号下 walk 最常走出的轨迹**（happy path），不是每次必走的固定模板。M2 在遍历阶段结果依赖地决定每一步，节点可跳过、可重复、可换序。

**典型轨迹**：情绪反应（M2 闸门）→ 经验式输出（M4 节点）→ 明确逻辑思考（C1 节点）→ 情绪输出（M2 收尾）

```
[阶段一 · 闸门] M2 本地评分
   ├─ 情绪第一反应（直出 · 不等 sub-agent）
   ├─ 价值 / 显著性判断
   └─ 初始遍历意图（起 walk 节点种子）
        │
        ▼
[阶段二 · 遍历] M2 walk 决策树，每节点拉一个 sub-agent
        │
   节点 · M4 ── 拉路径库 + 起手 / 桥接段
        │       （M2 判难 → 先出过渡语 "稍等想想 / 好问题"）
        ▼
   节点 · M3 ── 检索 R1 记忆库（强度由当前 walk 状态定）
        │       返回候选经验 + 类比 + 反例
        ▼
   节点 · C1 ── 接收 M3 候选作 priming + M4 桥接作延续点
        │       输出明确逻辑推理段（强度 walk 中最终定）
        ▼
   [M2 判够不够] 价值 + 逻辑够 → 收束 / 不够 → 继续 walk
        │
        ▼
[M2 整合输出] 收尾情绪 "就是这么个事" + 段落节奏映射 walk 轨迹
```

**节点次序不固定**：M4 与 M3 谁先谁后由初始遍历意图决定（他遇到信号习惯先调经验还是先出路径，是 IP 特化 · 详 `_main.md §D`）。上图只是一种常见排布。

**M4 第二反馈的功能**：在面对面对话中是思考空白时段填充器（让 C1 有深加工时间，对方不感空窗 = 持续在场感）。微信文字本身异步、用户不期待秒回，阶段 1 M4 真正功能 = 起手 / 过渡 / 句式提供，"填充"不需要。

## 跨件协同 trace 类型

### M2 → C1 跳变（结果依赖触发）

- 触发条件：难度高 / 价值冲突 / 反例出现 / M4 路径库无匹配 / 显著性突破阈值
- 跳变后 C1 工作强度由**当前 walk 状态**决定（不是闸门预定）
- C1 节点可跳过：若 walk 中发现是有现成路径的问题，M2 不进 C1 节点

### M3 → C1 引导（工程层 = 串行 priming）

- M2 在 walk 中先经过 M3 节点拿到候选，再进 C1 节点时把 candidates 作 C1 prompt prefix
- C1 受 M3 候选引导，不重启从头思考
- Loop Model 直觉桥梁机制（W11-W18 奠基 / 5-19 §H1）的工程层落地
- **机制层 vs 工程层**：机制层 M3 是"持续挂载实时给 C1 提交潜意识"；CC star 拓扑下 sub-agent 不能横向通信，工程层退化为 M2 串接（M3 节点返回 → M2 把候选喂给 C1 节点）。这条 trace 没断，只是穿过 M2 圆心。真正的并发引导是阶段 2+ 王磊 CLI 的差集需求。

### M4 → C1 桥接

- M4 节点先输出桥接段 → C1 prompt prefix 含 "M4 已输出: [桥接段]"
- C1 输出从 M4 收尾自然延续（不重启段落 / 不重申话题 / 不换语气）

### C1 → 过渡语切入（叫停归 M2）

- M2 在 walk 到 C1 节点前评估难度，难度高 → 调用 M4 过渡语库（"稍等想想 / 好问题"）先输出 → 再调 C1
- 用户感知 = 思考中

### C1 → M2 收尾回调

- C1 推理段后 M2 切回情绪输出收尾（"就是这么个事 / 对吧 / 是吧"）
- 整个 walk 闭环

### S1 ↔ 各 sub-agent

- S1 = 主 agent context window 自带（对话历史 + 当前 token），不独立建模
- S1 给各节点提供上下文；节点返回结果回填主 agent context（不写持久存储，只参与本轮）

## 动态化机制（不死板）

walk 不是刻板按固定路径生成。动态化来自两处：

### 节点权重 + 可跳过

每个节点是否进、进多深，由 M2 闸门初值 + walk 中精化共同决定：

| 信号难度 | 情绪反应 | M4 路径 | M3 检索 | C1 思考 |
|---|---|---|---|---|
| 极简（"你好"）| 弱 | 直接出招呼 | 跳过 | 跳过 |
| 简单 | 中 | 模板出 | 浅检 | 短 C1 |
| 中等 | 强 | 多模板候选 | 中检 | 中等 C1 |
| 高难度 | 强 | 桥接辅助 | 深检 + 多轮 | 全 C1 + 叫停 + 长段 |

### 结果依赖 + 偶发性

- 同一信号不同时候 walk 轨迹可不同（真实人的偶发性）
- 工程层：节点带 confidence + threshold + random seed（阶段 2+ LangGraph 条件边 + 动态 model selection）
