## 核心论点

**人的 skill = internalized alignment**（不可外部化的内部对齐），不是 externalized procedure（可写下的程序）。

蒸馏目标 = **还原一个具体的人在 Loop Model 五件解剖结构下的个人化配置 + 协同模式**。

商业产品 = **个人 IP 资产化**：
- **A 路** = 单一 IP 主的 AI 复刻 skill 包（形似上限 70-80%，scale 容易）
- **B 路** = 同一 IP 主的"流派传人"培养（神似 90-99%，scale 难单价高）
- 前置 trace 库共享 → A 先决（B = A 的 trace 库 + 内化引擎）

**竞品差异 / 护城河**：现行 AI 数字人 / IP 复刻产品（Character.AI 等）只拿**风格层 + 内容层 trace**——效果停在"听起来像但做事不像"。本项目用 Loop Model 拿**机制层 trace**——机制层稳定，风格 / 内容自动跟着出来。

## 五件功能分类

五件**不是平等同构**的子维度结构，分三类：

| 类 | 模块 | 角色 | 个人化 |
|---|---|---|---|
| (a) 路径结构主导 | M2 + M3 | IP 蒸馏核心 trace | 真个人化 |
| (b) 计算功能主导 | C1 + S1 | 方式偏好层 | C1 部分个人化 / S1 无 |
| (c) 短时封装层 | M4 | 实时封装 + 路径库 | 真个人化 |

**真个人化只在 M3 + M2**（C1 主体推理机制同构 · S007 启动注入），但 C1 工程上承担一部分偏好参数补 M3 不足（详 `sub_agent设计.md §C1 的个人化边界` + `sub_C1.md`）。S1 = LLM context window 自带，无个人化。

## M3 环路 · 检索整合（路径结构主导）

**功能**：从 R1 记忆库中调出相关历史 / 类比 / 案例并整合（不含写入）。

**通用 frame 内部组件**：语义记忆检索网络 + 情景记忆检索网络 + 整合输出。

**IP 特化子维度**：
- R1 记忆库索引偏好（专业内 / 跨学科 / 历史案例 / 私人体验的调用分布）
- 检索触发模式（关键词 / 结构相似 / 情绪共振）
- 类比映射偏好（隐喻源领域分布）
- 检索深度（浅近相关 vs 远距类比的偏好）
- 整合方式（序列拼接 / 矩阵对齐 / 抽象上升 / 反例对照）
- 核心 anchor 集 + 反复出现的 imagery + 概念关联结构（anchor 存储在 R1 记忆库层，不在 S1）

**环路 I/O**：input ← 当前情境 + walk 状态；output → M2（候选回应 + 类比框架）+ C1（priming）。

**控制流角色**：被 walk 到的检索节点 + 全系统统一数据服务。详 `M3_机制详解.md`。

---

## M2 环路 · 价值评估 / 取向调度（路径结构主导）

**功能**：信号的价值与显著性评估 + 调度其他模块 = orchestrator。

**通用 frame 内部组件**（Capability_Atlas v2 §十一 三层）：
- 表层 = OCEAN 5 + sub-facets
- 中层 = 4 维 adaptations（goals / interpretive schemes / coping strategies / 角色身份占位）
- 底层 = Schwartz 8 values + sub-facets

**IP 特化子维度**：
- value anchors 集（Schwartz values 配置 + 双极极性）
- trait 表达模式（OCEAN 5 + sub-facets 行为表达）
- adaptations 配置（领域内目标 + 解释 schema + 应对策略 + 身份 anchor）
- 显著性阈值（注意系统对什么敏感 / 什么默认过滤）
- 跨情境优先级（values 冲突解决规则）

**环路 I/O**：input ← 用户信号 + 各节点返回；output → 闸门三件（情绪 / 价值判断 / 初始意图）+ walk 调度（拉哪个 sub-agent + 强度）+ 整合输出。

**控制流角色**：orchestrator = 闸门 + walk 执行者（star 圆心）。详 `M2_机制详解.md`。

## C1 环路 · 显式判断（计算功能主导）

**功能**：A-consciousness 输出 + reframe + 反事实 + 空间运算。

**通用 frame 内部组件**：BA10l 调度 + DLPFC 缓冲 + IFG 控制 + Pre-SMA 编码 + IPS-SPL 空间运算。

**个人化边界**：主体推理机制同构（人人推理架构相同，深层差异是神经权重带来的强度、速度）。这决定 C1 不能当主 agent。但工程上 C1 承担一部分偏好参数——强度 / 速度 + 部分推理偏好（reframe / 反事实 / 逻辑性 / 升维），补 M3 引导之不足（LLM 原生推理会盖过 M3）。详 `sub_C1.md`。

**环路 I/O**：input ← M2 调度 + M3 priming + M4 桥接；output → M2（推理段）+ 主 agent context（结果回填）。

**控制流角色**：被 walk 到的推理节点 · 结果依赖触发 · 强度不预先定。详 `C1_机制详解.md`。

---

## M4 环路 · 实时短时封装 + 路径库（短时封装层）

**功能**：固有路径库（几十~几百条）匹配 + 第二反馈生成（情绪→逻辑渐变桥接段）+ 起手 / 过渡 / 收束语库。

**IP 特化子维度**：
- 固有路径库（信号→应答骨架抽象模式）
- 起手语 / 过渡语 / 收束语库（副语言层 · 从音频抓）
- 句式偏好 / 节奏

**环路 I/O**：input ← walk 状态 + 信号；output → M2（路径候选 + 桥接段 + 起手 / 过渡语）+ C1（桥接作延续点）。

**控制流角色**：被 walk 到的路径节点。详 `M4_机制详解.md`。

---

## S1 环路 · 心智空间运算（计算功能主导）

**功能（机制层）**：被动持续场 / P-consciousness 底物 / 当前心智空间的 **active workspace**——C1 在其上做 IPS-SPL 空间运算。

**关键边界**：S1 是**运算空间**不是**存储空间**——核心 anchor / 反复 imagery / 概念关联结构归 R1 记忆库层。

**工程层退化**：S1 = 主 agent context window 自带 buffer。运算能力分散到各 sub-agent 内部 context。"C1 → S1 更新"这条 trace 退化为"结果回主 agent context"，没断。无独立 sub-agent，无个人化。

**控制流角色**：context window buffer。详 `S1_机制详解.md`（极简）。

## 五件协同模式

控制流总览（两阶段：闸门 + 遍历）见 `sub_agent设计.md §M2 两阶段控制流`；典型 walk 轨迹 + 跨件 trace 见 `协同setting.md`。

**核心**：所有环路穿过 M2 圆心（star 拓扑）。baseline（无 C1 介入）= M2 闸门 + M4 / M3 浅节点即收束；C1 介入 = walk 中 M2 阈值突破 / M3 多候选冲突 / 反例触发，结果依赖地拉 C1 节点。

---

## 协同 setting = 独立 trace 类型

跨件协同**不能拆到五件内部**，是独立 trace：
- M2 → C1 跳变的触发条件
- M3 → C1 引导（priming）
- M4 → C1 桥接
- C1 → M2 收尾回调

详 `协同setting.md §跨件协同 trace 类型`。
